========================================
PASSO 1 -- CRIAR BOT TELEGRAM
========================================

1. Telegram, busca @BotFather
2. Manda: /newbot
3. Nome: ex "Assistente do Jonas"
4. Username (termina em bot): ex jonas_assistente_bot
5. Copia o TOKEN retornado

Pegar ID Telegram do dono:
1. Busca @userinfobot, manda qualquer msg
2. Copia o ID numerico (ex 1672601444)


========================================
PASSO 2 -- INSTALAR DEPENDENCIAS (1 COMANDO)
========================================

Linux (root):
   curl -fsSL https://raw.githubusercontent.com/denderson2013-bot/agente-openclaw-telegram-setup-alunos-denderson/main/bootstrap.sh | bash

Instala: Node 22, Python 3, ffmpeg, OpenClaw CLI 2026.4.24,
Claude Code CLI 2.1.81 (auth backup), tmux, PostgreSQL 16 + pgvector,
pnpm, pm2 globalmente.

Tempo: 5-10 min.


========================================
PASSO 3 -- ESCOLHER LLM E AUTENTICAR
========================================

Opcao A -- GLM 4.5/5 Turbo (Z.ai):
   1. openclaw configure
   2. Adicionar provider zai:
      - baseUrl: https://api.z.ai/api/coding/paas/v4
      - api: openai-completions
      - apiKey: <sua key>
   3. Definir modelo primary: zai/glm-5-turbo
   4. (Recomendado) Fallbacks: zai/glm-5.1, zai/glm-5

Opcao B -- GPT Codex 5.5 (OAuth via assinatura, SEM API key):
   1. openclaw configure
   2. Adicionar profile openai-codex (mode oauth)
   3. A CLI imprime uma URL no terminal. Copia essa URL.
   4. Cola no navegador do seu PC, JA LOGADO na sua conta ChatGPT Plus.
   5. Autoriza o acesso. O CLI captura o token automaticamente e salva. Toda chamada vai consumir da sua quota da assinatura.
   6. Definir modelo primary: openai-codex/gpt-5.5
   7. (Recomendado) Fallback: anthropic/claude-opus-4-6

Anthropic (auth secundaria, opcional):
   claude auth login --claudeai
   (so se quiser usar claude-opus-4-6 como fallback)


========================================
PASSO 4 -- CONFIGURAR CANAL TELEGRAM
========================================

Adiciona o canal Telegram via openclaw configure:

   openclaw channels add telegram \
     --bot-token <TOKEN_DO_BOTFATHER> \
     --allow-from <SEU_USER_ID_TELEGRAM>

Ou edita /root/.openclaw/openclaw.json manualmente, secao channels.telegram:

   "telegram": {
     "enabled": true,
     "dmPolicy": "allowlist",
     "botToken": "<TOKEN>",
     "allowFrom": ["<USER_ID>"],
     "groupPolicy": "allowlist",
     "streaming": {"mode": "partial"},
     "actions": {"reactions": true}
   }

========================================
COMANDOS DO DIA A DIA
========================================

Status geral:
   systemctl status openclaw-gateway
   openclaw health
   openclaw doctor

Logs ao vivo:
   journalctl -u openclaw-gateway -f
   openclaw logs -f

Restart:
   systemctl restart openclaw-gateway

Config:
   openclaw config get gateway.port
   openclaw config set agents.defaults.model.primary zai/glm-5-turbo
   openclaw configure          # interativo
   nano /root/.openclaw/openclaw.json && systemctl restart openclaw-gateway

Adicionar/remover canal:
   openclaw channels add telegram --bot-token XXX
   openclaw channels remove discord

Ver agentes:
   openclaw agents list
   openclaw agents show paulo

Memoria:
   openclaw memory search "ultimo lancamento"
   openclaw memory reindex

========================================
PROBLEMAS COMUNS
========================================

"openclaw doctor falha em telegram"
   Provavel token errado ou bot bloqueado. Confere:
   openclaw config get channels.telegram.botToken
   Refaz com /token no @BotFather se precisar.

"systemd nao sobe gateway"
   journalctl -u openclaw-gateway -n 50
   Maioria das vezes e config invalido. Restaura backup:
   cp /root/.openclaw/openclaw.json.last-good /root/.openclaw/openclaw.json
   systemctl restart openclaw-gateway

"GLM retorna erro 401/403"
   API key Z.ai expirada ou sem credito. Recarrega no painel.

"GPT Codex pede pra logar de novo"
   OAuth token expirou. Roda:
   openclaw configure
   Refaz o login.

"Subagente nao responde"
   Verifica que ele esta no allowAgents do main:
   openclaw config get agents.list[0].subagents.allowAgents
   Adiciona se faltar.

"VPS reboot e nao volta"
   systemctl is-enabled openclaw-gateway
   Tem que dar enabled. Se der disabled:
   systemctl enable openclaw-gateway
