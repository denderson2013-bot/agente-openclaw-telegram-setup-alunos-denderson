========================================
COMO INSTALAR OpenClaw ORIGINAL PURO + TELEGRAM v3
(Suporta GLM 4.5/5 Turbo OU GPT Codex 5.5)
Validado ao vivo em Ubuntu 25.10 + OpenClaw 2026.6.5
========================================

Repositorio:
https://github.com/denderson2013-bot/agente-openclaw-telegram-setup-alunos-denderson

Instala o OpenClaw original puro (https://github.com/openclaw/openclaw),
sem customizacao. Agente generico, com o nome que voce escolher.

O que voce tem no final:
- Agente OpenClaw (ex: Bia-bot) conversando via Telegram
- Backend LLM: GLM (Z.ai) OU GPT Codex 5.5
- Gateway OpenClaw rodando 24/7 via systemd
- (Opcional) audio: Whisper entrada / ElevenLabs saida


========================================
PRE-REQUISITOS
========================================

- VPS Ubuntu 22.04+ ou macOS 13+ (2-4 GB RAM)
- Backend LLM (uma das opcoes):
  * GLM: API key Z.ai paga
  * GPT Codex: assinatura ChatGPT Plus ativa (sem API key, login OAuth via navegador)
- Conta Telegram (@BotFather + @userinfobot)
- (Opcional) OPENAI_API_KEY (audio entrada via Whisper)
- (Opcional) ELEVENLABS_API_KEY (audio saida)


========================================
PASSO 1 -- CRIAR BOT TELEGRAM
========================================

1. Telegram, busca @BotFather
2. Manda: /newbot
3. Nome: ex "Assistente do Joao"
4. Username (termina em bot): ex joao_assistente_bot
5. Copia o TOKEN retornado

Pegar seu ID Telegram:
1. Busca @userinfobot, manda qualquer msg
2. Copia o ID numerico (ex 123456789)


========================================
PASSO 2 -- INSTALAR DEPENDENCIAS + OpenClaw (1 COMANDO)
========================================

Linux (root):
   curl -fsSL https://raw.githubusercontent.com/denderson2013-bot/agente-openclaw-telegram-setup-alunos-denderson/main/bootstrap.sh | bash

Instala: Node 22 (via nvm), Python 3, ffmpeg, tmux e o
OpenClaw CLI ORIGINAL (npm install -g openclaw@latest).

Tempo: 5-10 min.


========================================
PASSO 3 -- CONFIGURAR O TELEGRAM (config patch, NAO na mao)
========================================

ATENCAO 2026.6.5: NAO escreva o openclaw.json na mao (da
"OpenClaw config is invalid: <root>: Invalid input"). Usa config patch,
que valida na escrita. dmPolicy/groupPolicy sao enums obrigatorios.

   cat > /tmp/oc-telegram.json <<JSON
   {
     "channels": {
       "telegram": {
         "enabled": true,
         "botToken": "<TOKEN_DO_BOTFATHER>",
         "dmPolicy": "allowlist",
         "groupPolicy": "allowlist",
         "allowFrom": ["<SEU_USER_ID>"]
       }
     }
   }
   JSON
   openclaw config patch --file /tmp/oc-telegram.json
   rm -f /tmp/oc-telegram.json

   # OBRIGATORIO: sem gateway.mode o gateway NEM SOBE (exit 78)
   openclaw config set gateway.mode local
   openclaw config validate            # tem que dizer "Config valid"

Enums: dmPolicy = pairing|allowlist|open|disabled ; groupPolicy = open|disabled|allowlist.
Com allowlist nos dois + seu user_id em allowFrom voce fala com o bot direto.
Nome do agente (agent.name) NAO existe na 2026.6.5 -- roda sem nome custom (VERIFICAR via 'openclaw config schema').


========================================
PASSO 4 -- ESCOLHER LLM E AUTENTICAR
========================================

Opcao A -- GLM 4.5/5 Turbo (Z.ai, API key):
   cat > /tmp/oc-zai.json <<JSON
   {"models":{"providers":{"zai":{
     "baseUrl":"https://api.z.ai/api/coding/paas/v4",
     "apiKey":"<SUA_KEY>",
     "api":"openai-completions",
     "models":[{"id":"glm-5-turbo","name":"GLM-5-Turbo","input":["text"],"contextWindow":204800,"maxTokens":131072}]
   }}}}
   JSON
   openclaw config patch --file /tmp/oc-zai.json && rm -f /tmp/oc-zai.json
   openclaw models set zai/glm-5-turbo

Opcao B -- GPT Codex 5.5 (OpenAI, OAuth via assinatura, SEM API key):
   openclaw models auth login --provider openai --device-code
   (o CLI imprime a URL https://auth.openai.com/codex/device + um Code, ex EDYT-MLUDG;
    abre a URL no PC JA LOGADO no ChatGPT Plus, digita o code, autoriza;
    o CLI detecta sozinho e imprime "OpenAI device code complete")
   openclaw models set openai/gpt-5.5

   openclaw config validate            # "Config valid" antes de subir o gateway

(Atalhos: scripts/configure-glm.sh e scripts/configure-gpt-codex.sh)


========================================
PASSO 5 -- SUBIR O GATEWAY (systemd 24/7)
========================================

   curl -fsSL https://raw.githubusercontent.com/denderson2013-bot/agente-openclaw-telegram-setup-alunos-denderson/main/systemd/openclaw-gateway.service -o /etc/systemd/system/openclaw-gateway.service
   # troca o placeholder do binario pelo caminho real e o nome na Description
   sed -i "s#__OPENCLAW_BIN__#$(command -v openclaw || echo /usr/local/bin/openclaw)#" /etc/systemd/system/openclaw-gateway.service
   sed -i "s/AGENTE_NAME_PLACEHOLDER/OpenClaw/" /etc/systemd/system/openclaw-gateway.service
   systemctl daemon-reload
   systemctl enable --now openclaw-gateway
   systemctl status openclaw-gateway

========================================
COMANDOS DO DIA A DIA
========================================

Status:
   systemctl status openclaw-gateway
   openclaw gateway status
   openclaw doctor

Logs ao vivo:
   journalctl -u openclaw-gateway -f

Restart:
   systemctl restart openclaw-gateway

Modelos:
   openclaw models list
   openclaw models set openai/gpt-5.5     (ou zai/glm-5-turbo)
   openclaw models status --probe

Config:
   nano /root/.openclaw/openclaw.json && systemctl restart openclaw-gateway


========================================
PROBLEMAS COMUNS
========================================

"openclaw onboard trava numa VPS headless"
   E por isso que esse setup NAO usa o onboard interativo. Usa os passos
   controlados acima (config + models set + systemd).

"systemd nao sobe gateway"
   journalctl -u openclaw-gateway -n 50 --no-pager
   - "missing gateway.mode" (exit 78) -> openclaw config set gateway.mode local
   - "Invalid input" -> JSON escrito na mao; refaz via openclaw config patch + openclaw config validate
   - placeholder __OPENCLAW_BIN__ sobrando no unit -> refaz o sed do PASSO 5
   Roda sempre: openclaw config validate && openclaw doctor antes de desistir.
   Restaura backup se preciso:
   cp /root/.openclaw/openclaw.json.last-good /root/.openclaw/openclaw.json
   systemctl restart openclaw-gateway

PERSISTENCIA: erro nao e parada. Diagnostica (config validate + doctor),
corrige e continua. So termina quando o bot responder de verdade no Telegram.

"GLM retorna 401/403"
   API key Z.ai expirada ou sem credito.

"GPT Codex pede pra logar de novo"
   openclaw models auth login --provider openai --device-code

"openclaw: command not found"
   npm global bin fora do PATH:
   export PATH="$(npm config get prefix)/bin:$PATH"
   (ou reroda o bootstrap, que cria symlink em /usr/local/bin)

"VPS reboot e nao volta"
   systemctl is-enabled openclaw-gateway  -> tem que dar enabled
   Se nao: systemctl enable openclaw-gateway
